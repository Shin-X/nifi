This processor is designed to take any valid JSON and index it into Elasticsearch 5.x or newer. Unlike the record-aware alternative, it only does index operations, not updates, upserts or deletes.

The ID for each document can be set in one of three ways:

  • FlowFile attribute - only possible if the flowfile contains only a single document. Arrays of documents will cause an error to be raised.
  • JsonPath operation - searches within the document for a particular field.
  • Default - Elasticsearch will generate one. This will be used when neither the attribute nor JsonPath configuration fields are used.

Example document:

        {
            "index": "messages",
            "type": "message",
            "message": "Hello, world",
            "from": "john.smith"
        }

The following JsonPath operations will extract the index and type:

  • $.index
  • $.type
